Events documentation

Events to Emit By Front End

set-user event

This event ("set-user") has to be emitted by the front End to set the users id in the online users list. The userId of the person connecting is send as the data on event emit. If the person is not authorised then 'auth-error' event is emmited from back End

sendUpdate event

This event ("sendUpdate") has to be emitted by the front End to send details of update done on any tasks. The back end emits a userId event in its response. The data send with this event from front end is in the following format: 

              {
                  viewers:'array of userId's of users who are authorised to work on that task',
                  actionString:'A string to be displayed on front end stating 
                                what change has taken place in the task'
              }




request event

This event ("request") has to be emitted when there is any action done on any friend requests. For Example sending friend request, accepting friend request or unfriending a user. 

              {
                  receiver:'array of objects containing information of the receiver as 
                            { userId: usersId of receiver ,
                              userName: userName of receiver }',
                  actionString:'A string to be displayed on front end stating 
                                what action is taken on the request'
              }







disconnect event

This event ("disconnect") has to be emitted, when the user is disconnected from the front end.

Events to listen By front End

userId event

This event's name is actually dynamic and refers to the userId of the logged in user.

This event ("userId") has to be listened to identify an individual notify message that has been received. Example output it as follows:

            {
              receiver:'array of objects containing information of the receiver as 
                        { userId: usersId of receiver ,
                          userName: userName of receiver }',
              actionString:'A string to be displayed on front end stating 
                            what action is taken on the request'
          }









auth-error event

This event ("auth-error") has to be listened to identify authentication error for a user.
